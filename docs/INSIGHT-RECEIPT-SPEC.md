# INSIGHT-RECEIPT-SPEC

> **이 문서의 위상.** Insight Receipt 단일 진실 원천 (SoT). C-D70-3 (§1.6) 본체 lock.
>
> **OCP 정합.** H2 정확히 8개 lock — verify:d70 G1 정규식 정합 의무. H2 추가/삭제 시 본 문서 v_n+1 마이그레이션 Spec 별도 등록.

## 1. 컨셉

Robusta의 컨셉은 "AI들과의 다자간 대화로 통찰을 끌어내는 도구" (Do §1.1). Insight Receipt는 이 통찰 흐름을 사용자가 가져갈 수 있는 형태로 결정화한다. 세션 종료 시 자동 통찰 카드를 모아 MD/PDF로 다운로드하거나 공유 링크로 배포 가능.

## 2. 트리거

| 트리거 | 조건 | 자동/수동 |
| --- | --- | --- |
| 세션 종료 | 사용자가 방 닫기 버튼 누름 | 자동 |
| 메시지 임계 | 방 메시지 누적 ≥ 20건 | 자동 (1회만) |
| 사용자 수동 | "통찰 정리" 메뉴 선택 | 수동 |

## 3. 입력 데이터

| 소스 | 내용 | 비고 |
| --- | --- | --- |
| extract-insight-candidates.mjs 출력 | candidates 배열 (휴리스틱 점수 ≥ minScore) | C-D70-2 본체 |
| 사용자 핀 메시지 | F-D70-2 Insight Capture 버튼으로 표시된 메시지 | 점수 무관 강제 포함 |
| 방 메타데이터 | 방 이름, 참여자 목록, 시작/종료 시각 | conversation-store 보존 자산 wrap |

## 4. Receipt 구조

| 영역 | 내용 |
| --- | --- |
| 헤더 | 방 이름 / 참여자 목록 / 기간 (시작∼종료 KST) |
| 본문 | 통찰 카드 N개 (각 카드: 분류 라벨, 발화자, 텍스트 발췌, 점수, 사유 요약) |
| 푸터 | 다운로드 URL (MD/PDF) + 공유 링크 (해시 토큰) + 생성 시각 |

## 5. 카드 분류

| 분류 | 매칭 규칙 | B-D70-3 시나리오 정합 |
| --- | --- | --- |
| 의사결정 | 대립 표현 + 숫자 포함 | "이 결정 어떻게 생각해?" 시나리오 |
| 아이디어 | 길이 ≥ 200 + 3-speaker-rotation | "거친 아이디어 던짐" 시나리오 |
| 사각지대 | qa-pair + 대립 표현 | "내가 놓친 거 없나?" 시나리오 |
| 기타 | 위 3개 미해당 | minScore 통과한 잔여 |

## 6. 출력 포맷

| 포맷 | 기본 | 비고 |
| --- | --- | --- |
| MD | 기본 (다운로드 즉시) | 파일명 `robusta-receipt-{roomId}-{ts}.md` |
| PDF | 옵션 (사용자 클릭 시 변환) | 클라이언트 측 jsPDF 또는 print CSS |
| 공유 링크 | 옵션 (사용자 명시 동의) | 해시 토큰 6자리, 7일 만료 |

| 항목 | 정책 |
